# 鼠肝 FL 实验设计 — 聚合策略对比

> 创建：2026-07-04
> 背景：冬生提出顺序链式 FL + 性能门控，需要设计实验验证哪条路更好

## 实验维度

FL 聚合策略有三个正交维度，每个维度有多种选择：

### 维度 A：聚合时机
| 选项 | 说明 | 信息流 |
|------|------|--------|
| **并行同步** | 所有节点基于同一全局模型训练，训练完统一聚合 | 下一轮才能受益于其他节点 |
| **顺序链式** | 节点1→2→3 串行，后一个用前一个更新后的模型 | 当轮即可受益 |
| **混合** | 节点1→2 顺序，节点3 并行加入 | 折中 |

### 维度 B：更新策略（性能门控）
| 选项 | 说明 | 风险 |
|------|------|------|
| **无条件聚合** | FedAvg/EWA/FedProx，不管性能 | 噪声节点拖低全局 |
| **硬门控替换** | 性能更好才更新全局，否则保留旧参数 | 可能陷入局部最优 |
| **软门控加权** | EWA 按性能加权，差节点权重低 | 仍有噪声但被压制 |
| **本地门控存储** | 各节点收到全局参数后，本地比较决定是否存储 | 节点自治，全局可能不收敛 |

### 维度 C：节点顺序（仅顺序链式）
| 选项 | 说明 | 理由 |
|------|------|------|
| **B1→B2→B3** | 好的先跑 | 高质量参数先传播 |
| **B3→B2→B1** | 差的先跑 | 测试弱节点起步的影响 |
| **质量排序** | 按 mAP 动态排序 | 自适应 |
| **随机** | 每轮随机 | 对照组 |

### 维度 D：评估基准
| 选项 | 说明 | 问题 |
|------|------|------|
| **统一 val_set** | 9 张（B1/B2/B3 各 3 张） | 公平但小 |
| **本地 val** | 各节点用自己的 val | 不可比 |
| **交叉评估** | 全局模型在三个节点 val 上分别评估 | 信息更丰富 |

## 顺序链式 FL 详细设计（E4-E8）

### 流程
```
每轮:
  global_model = init_model
  for node in [B1, B2, B3]:  # 顺序
    local_model = train(global_model, node_data)
    local_mAP = eval(local_model, val_set)
    if 策略 == "无门控":
      global_model = aggregate(global_model, local_model)
    elif 策略 == "硬门控":
      if local_mAP > global_mAP:
        global_model = local_model  # 替换
      # else: 保留旧 global_model
    elif 策略 == "软门控":
      w = compute_ewa_weight(local_mAP, global_mAP)
      global_model = w * local_model + (1-w) * global_model
    elif 策略 == "本地门控":
      # 广播 global_model, 各节点自己决定是否存储
      broadcast(global_model)
  
  # 轮末: 各节点收到最终 global_model
  for node in [B1, B2, B3]:
    node_model = eval(global_model, node_local_val)
    node_stored = node_model if better else keep_local
```

### 关键设计决策
1. **聚合方式**：无门控用 FedAvg（按数据量加权），硬门控直接替换
2. **性能比较**：统一 val_set（9 张），公平可比
3. **训练轮数**：10 rounds × 10 local epochs（和 Phase 1 一致）
4. **节点顺序**：默认 B1→B2→B3（高质量先行），E7 反序做对照

## 预期分析

### 对比指标
- 全局 mAP50/mAP50-95 收敛曲线
- 每轮每个节点训练后的 mAP 变化
- 最终全局模型在统一 val_set 上的 P/R/mAP
- 检测可视化（绿框 vs GT 红框）

### 核心问题
1. **顺序 vs 并行**：E4 vs E1（都是无门控）→ 信息流差异
2. **门控 vs 无门控**：E5 vs E4（都是顺序）→ 噪声过滤
3. **硬 vs 软门控**：E5 vs E6 → 门控强度
4. **节点顺序**：E5 vs E7 → 顺序敏感性
5. **统一 vs 本地门控**：E5 vs E8 → 评估基准选择

### 统计可靠性
- val_set 只有 9 张（B1/B2/B3 各 3 张），mAP 波动大
- 每个实验跑 3 次取均值+标准差
- 或增大 val_set（但数据量有限）

## 决策点（需冬生确认）

1. **Phase 2 先跑哪几个？** E4+E5+E6（三组核心对比）还是全部？
2. **硬门控的"性能更好"阈值？** 严格 > 还是 ≥？是否需要 margin（如 +1%）？
3. **软门控的信号？** mAP50-95（和 EWA 一致）还是 mAP50？
4. **是否需要跑 3 次取均值？** 时间成本 vs 统计可靠性
